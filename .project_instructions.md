# Enxoval Anna Clara - Landing Page Project

## Project Overview
Uma landing page elegante e moderna para celebrar o enxoval da bebê Anna Clara, desenvolvida com Next.js 14, React 18, Tailwind CSS e Framer Motion. A aplicação permite que pessoas escolham presentes de uma lista de 24 itens ou contribuam via PIX com upload de comprovante.

## Technical Stack
- **Framework:** Next.js 14 com App Router
- **Frontend:** React 18, TypeScript
- **Styling:** Tailwind CSS 3 com gradientes pastéis
- **Animations:** Framer Motion
- **Database:** PostgreSQL (Prisma ORM)
- **File Upload:** AWS S3 (presigned URLs)
- **Icons:** Lucide React
- **Form Validation:** Native HTML5 + custom validation

## Design System
- **Theme:** Arco-íris em tons pastéis
- **Primary Colors:**
  - Pink: #FFB3D9
  - Blue: #B3D9FF
  - Yellow: #FFFACD
  - Green: #B3FFB3
  - Purple: #D9B3FF
- **Shadows:** Elegant (subtle) and Elevated (stronger)
- **Border Radius:** 1-2xl rounded corners
- **Typography:** Inter font, sans-serif

## Key Features Implemented

### 1. Hero Section
- Animações de entrada elegantes com stagger delay
- Apresentação clara de propósito
- Data prevista do parto (12 de junho de 2026)
- CTAs destacados com gradientes

### 2. Como Funciona
- 4 cards com ícones animados
- Explicação clara do fluxo: Escolher → Registrar → Confirmar → Pronto
- Hover effects sofisticados
- Responsivo em mobile/desktop

### 3. Formulário Elegante
- Campos obrigatórios: Nome e WhatsApp
- Validação em tempo real
- Mensagens de sucesso/erro animadas
- Design minimalista com foco

### 4. Grid de Itens (24 itens)
- Layout dinâmico com cores pastéis rotativas
- Desabilita itens já escolhidos (via API)
- Feedback visual de seleção
- Atualização em tempo real (polling 5s)

### 5. Módulo PIX
- Dropdown com 6 valores predefinidos (R$50-R$500)
- Upload de comprovante com preview de imagem
- Validação: tipo (PNG/JPG/GIF) e tamanho (máx 5MB)
- Upload direto para S3 via presigned URLs

### 6. Tabela de Participantes
- Exibe nome, WhatsApp e presente/valor PIX
- Ordenação por data de registro (mais recente primeiro)
- Animação de entrada de linhas
- Hover effects elegantes
- Contador de participantes

### 7. PIX Counter
- Animação "count-up" do total arrecadado
- Atualização em tempo real via polling
- Design destacado com gradiente
- Formatação monetária em Reais

## Database Schema
```prisma
model Participant {
  id        String   @id @default(cuid())
  name      String
  whatsapp  String
  itemName  String?  // Item escolhido (se gift)
  pixValue  Int?     // Valor em centavos (se PIX)
  receiptPath String? // Caminho S3 do comprovante
  createdAt DateTime @default(now())
  
  @@index([createdAt])
}
```

## API Routes
- `POST /api/register` - Registrar novo participante
- `GET /api/participants` - Listar todos participantes
- `GET /api/chosen-items` - Listar itens já escolhidos
- `GET /api/pix-total` - Total arrecadado em PIX
- `POST /api/upload/presigned` - Gerar URL presigned para upload

## Responsive Design
- Mobile-first approach
- Breakpoints: sm (640px), md (768px), lg (1024px), xl (1280px)
- Grid fluido: 1 col (mobile) → 2 cols (tablet) → 3-4 cols (desktop)
- Padding e margens responsivos

## Animation Patterns
- Fade-in on scroll com `whileInView`
- Stagger children com delay progressivo
- Hover effects com scale/y transforms
- Smooth transitions com duration 0.3-0.8s
- Count-up animation para números

## Performance Optimizations
- Next.js Image component para otimização
- Presigned URLs para upload direto (sem proxy)
- Polling inteligente (5s interval) em vez de WebSocket
- CSS-in-motion (Framer) apenas no client
- Server components por padrão

## Important Notes
- Todos os componentes interativos usam 'use client'
- S3 uploads são privados (sem ACL público)
- Presigned URLs têm expiração de 1 hora
- FileReader para preview antes de upload
- Crash prevention com optional chaining (?.)
- Hydration-safe com suppressHydrationWarning onde necessário
